Diff Logic Implementation

How the four toggles affect JSON comparison

Toggle Behavior
1. Pretty Print

Parse JSON → Format with 2-space indent → Compare formatted strings

2. Ignore Whitespace

Normalize whitespace (collapse multiple spaces, trim lines) → Compare

3. Ignore Key Order

Custom logic: Parse objects → Sort keys recursively → Compare structure

4. Semantic Type Diff

Deep comparison: string "123" vs number 123 → treat as same type

Key Implementation Details

  • CodeMirror Integration: Minimal setup - just line numbers, no heavy IDE features. Editor state managed as simple string values.
  • Shiki Highlighting: Use `shiki` for JSON syntax highlighting. Works in RSC, tree-sitter based.
  • Diff Library: Use `diff` npm package for text/word/line diffing with custom preprocessing for JSON.
  • Key Order Logic: Custom recursive function to sort object keys before comparison.
  • Semantic Type: Deep comparison that coerces types (string "123" ↔ number 123, boolean ↔ string "true").

Why This Approach

  • Diff library handles text/word/line efficiently
  • Custom JSON preprocessing gives us full control
  • Minimal CodeMirror = fast load, small bundle
  • Shiki is lightweight vs Prism/Highlight.js
  • Hook-based logic = easy to test in isolation

Trade-offs

  • Custom key-sort logic = more code to maintain
  • Semantic type coercion can hide real differences
  • Large files may need performance optimization
  • CodeMirror/Shiki integration needs care
