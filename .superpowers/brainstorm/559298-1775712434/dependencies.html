Dependencies & Package Setup

Required npm packages and why we need them

Core Dependencies
Package Version Purpose
diff ^7.0.0 Text/word/line diffing (unified diff output)
@codemirror/view ^6.28.0 Minimal editor (line numbers only)
@codemirror/state ^6.4.0 Editor state management
@codemirror/commands ^6.6.0 Basic editor commands (history, selection)
@codemirror/language ^6.10.0 Language support (for future XML/CSV)
@codemirror/autocomplete ^6.18.0 Bracket matching (lightweight)
shiki ^1.10.0 Syntax highlighting (tree-sitter based, RSC compatible)

Why This Stack?

Advantages

  • CodeMirror 6: Modular, tree-sitter based, lighter than Monaco/Ace
  • Minimal setup: Only line numbers, no heavy IDE features
  • Shiki: VS Code's highlighter, works in RSC, very accurate
  • diff package: Battle-tested, handles edge cases well
  • Bundle size: ~150KB gzipped vs ~500KB for full editor

Trade-offs

  • More packages to install vs single editor library
  • CodeMirror 6 has learning curve (different from v5)
  • Shiki async API needs careful handling
  • Less built-in vs Monaco (but we don't need it)

Installation Command

npm install diff @codemirror/view @codemirror/state @codemirror/commands @codemirror/language @codemirror/autocomplete shiki

Future Additions (Not Now)

XML support: No additional packages needed (custom tree-walker)
CSV support: papaparse (when we add CSV format)
YAML support: yaml package (if needed later)
