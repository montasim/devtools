State Management

How data flows through the component

EditorPane State Structure
interface EditorPaneState {
  // Content
  leftContent: string;           // Raw JSON in left pane
  rightContent: string;          // Raw JSON in right pane

  // Validation
  leftValid: boolean;            // Is left JSON valid?
  rightValid: boolean;           // Is right JSON valid?
  leftError: ParseError | null;  // Error details (line, column, message)
  rightError: ParseError | null;

  // Diff state
  diffResult: DiffResult | null; // Computed diff
  isComputing: boolean;          // Loading state

  // UI state
  showLineNumbers: boolean;      // Always true (for future)
  leftEditorFocused: boolean;    // For keyboard shortcuts
  rightEditorFocused: boolean;
}
Props Interface
interface EditorPaneProps {
  // Toggle states (from Toolbar)
  ignoreKeyOrder: boolean;
  prettyPrint: boolean;
  ignoreWhitespace: boolean;
  semanticTypeDiff: boolean;

  // Initial content (optional)
  initialLeftContent?: string;
  initialRightContent?: string;

  // Event handlers
  onCompare?: (result: DiffResult) => void;
  onError?: (error: Error) => void;

  // Styling
  className?: string;
}
State Updates
User types in left editor
1. onChange → setLeftContent()
2. validateJson() → setLeftValid/Error
3. Both valid? → Enable Compare button
User clicks Compare
1. setIsComputing(true)
2. useJsonDiff(content, toggles)
3. setDiffResult(result)
4. setIsComputing(false)
5. onCompare?.(result)
User uploads file
1. FileReader.readAsText()
2. validate → setContent()
3. Error? → toast notification
Data Flow Diagram
JsonEditor onChange EditorPane
EditorPane validates JSON.parse()
EditorPane calls useJsonDiff
useJsonDiff returns DiffResult
EditorPane passes to DiffPanel

Why This Approach?

  • Lifted state: EditorPane owns all state, children are presentational
  • Controlled inputs: JsonEditors are controlled components
  • Validation separation: Parse errors stored separately from content
  • Loading states: isComputing prevents duplicate diff requests
  • Event callbacks: Optional props for parent component integration
