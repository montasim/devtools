Testing Strategy

How we'll verify the EditorPane works correctly

Unit Tests
useJsonDiff Hook
  • Toggle combinations (16 scenarios)
  • Valid JSON comparisons
  • Invalid JSON handling
  • Empty content edge cases
JsonEditor Component
  • Renders CodeMirror instance
  • onChange callback fires
  • Error state displays correctly
  • File upload triggers onChange
DiffPanel Component
  • Renders unified diff correctly
  • Line numbers match content
  • Color coding is accurate
  • Empty state shows message
Integration Tests
EditorPane Full Flow
  • User types in both editors
  • Validation states update
  • Compare button enables/disables
  • Click Compare → diff appears
  • Toggles change diff result
Error Scenarios
  • Invalid JSON shows error
  • Invalid file upload rejected
  • Diff failure displays error
  • Huge file warning shows
Keyboard Navigation
  • Tab between editors
  • Ctrl+Enter triggers Compare
  • Escape clears errors

Test Cases - useJsonDiff Hook

Scenario Input Toggles Expected
Identical JSON Same in both Any No differences
Key order differs {a:1,b:2} vs {b:2,a:1} Ignore Key Order: ON No differences
Whitespace differs Format differs Ignore Whitespace: ON No differences
Type coercion "123" vs 123 Semantic Type: ON No differences
Real difference {a:1} vs {a:2} Any Show difference

Test Tools

  • Framework: Vitest (already in project for Next.js)
  • React Testing: @testing-library/react
  • Assertions: Vitest built-in + custom matchers
  • Mocking: Vitest vi.mock() for CodeMirror, Shiki
  • Coverage: Vitest --coverage (aim for 80%+)

Manual Testing Checklist

Editor Interactions

Type, paste, upload files, delete content, use keyboard shortcuts

Validation

Invalid JSON shows error, valid JSON enables Compare, errors clear on fix

Diff Toggles

Each toggle changes result, combinations work correctly

Performance

Large files (1MB+) don't freeze UI, diff completes in <2s

Success Criteria

  • ✅ All unit tests pass (useJsonDiff, JsonEditor, DiffPanel)
  • ✅ Integration tests cover full user flow
  • ✅ Manual testing checklist complete
  • ✅ Build succeeds with no TypeScript errors
  • ✅ Bundle size increase <200KB gzipped
  • ✅ No console errors in browser
